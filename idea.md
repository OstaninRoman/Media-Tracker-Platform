# Media Tracker Platform — Идея

## Концепция

Платформа для отслеживания потреблённого медиа-контента: фильмы, сериалы, книги. Пользователь может вести учёт того, что посмотрел/прочитал, оценивать и оставлять заметки.

## Основные сущности

### Пользователь
- Email (уникальный)
- Пароль (хешированный)
- Дата регистрации
- Профиль (опционально: аватар, био)

### Медиа-элементы

**Фильм**
- Название (RU/EN)
- Год выпуска
- Жанры (список)
- Постер (URL)
- Описание
- Режиссёр
- Оценка пользователя (1-10)
- Статус (просмотрен / в планах / бросил)
- Заметки пользователя
- Дата просмотра

**Сериал**
- Название
- Год начала/конца
- Жанры
- Постер
- Описание
- Количество сезонов
- Оценка
- Статус (смотрю / просмотрен / бросил)
- Текущий сезон/серия
- Заметки

**Книга**
- Название
- Автор(ы)
- Год издания
- Жанры
- Обложка (URL)
- Описание
- Количество страниц
- Оценка
- Статус (прочитана / читаю / бросил)
- Заметки
- Дата прочтения

## Пользовательские фичи

- [ ] Регистрация / вход
- [ ] Добавление фильма/сериала/книги в свой список
- [ ] Оценка (1-10 звёзд)
- [ ] Статус (просмотрено/в планах/бросил)
- [ ] Заметки к каждому элементу
- [ ] Фильтрация по статусу / жанру / оценке
- [ ] Поиск по названию
- [ ] Статистика (сколько просмотрел за год, любимые жанры)

### Стек технологий

**Frontend:**
- JavaScript + React (или Vue/Svelte)
- SPA подход (Single Page Application)
- TailwindCSS или обычный CSS

**Backend:**
- Python 3.10+
- FastAPI (современный, быстрый, auto-docs)
- Pydantic для валидации данных
- SQLAlchemy (ORM)

**Database:**
- PostgreSQL (реляционная, надёжная)
- SQLite (для старта/прототипа)

**Аутентификация:**
- JWT токены
-bcrypt для хеширования паролей

## API (REST)

### Auth
- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход
- `GET /api/auth/me` — текущий пользователь

### Films
- `GET /api/films` — список фильмов пользователя
- `POST /api/films` — добавить фильм
- `PUT /api/films/{id}` — обновить
- `DELETE /api/films/{id}` — удалить

### Series / Books — аналогично

## Структура базы данных (PRD)

```sql
users
  - id (PK)
  - email (unique)
  - password_hash
  - created_at

films
  - id (PK)
  - user_id (FK)
  - title
  - year
  - genre (array)
  - poster_url
  - description
  - director
  - rating (nullable)
  - status (enum: planned, watched, dropped)
  - notes
  - watched_date

series (same structure + seasons, current_episode)
books (same structure + author, pages)
```

## Принятые решения

### Регистрация
- Да, по email + пароль
- JWT токены для аутентификации
- bcrypt для хеширования паролей

### Источники данных о медиа
- **Фильмы/Сериалы:** TMDB API (The Movie Database) — бесплатный, огромная база
- **Книги:** Open Library API — полностью открытый
- Это легально и не нарушает авторские права

### Архитектура
```
Frontend:  React (SPA)
Backend:   FastAPI (Python)
Database:  PostgreSQL
Auth:      JWT
Hosting:   Railway / Render (free tier)
```

### Почему такой выбор
- SPA даёт современный UX (как мобильное приложение)
- FastAPI — быстрый и современный Python framework
- PostgreSQL — надёжная реляционная БД
- TMDB/Open Library — бесплатные API с хорошим покрытием
