## ПОШАГОВЫЙ ПРОЦЕСС

### Шаг 3.1: Построение OST (Opportunity Solution Tree)

Построй OST по методологии Teresa Torres:

```
Desired Outcome (желаемый результат)
     │
     ├── Opportunity 1
     │       ├── Solution 1.1
     │       ├── Solution 1.2
     │       └── Solution 1.3
     │
     ├── Opportunity 2
     │       ├── Solution 2.1
     │       └── Solution 2.2
     │
     └── ... и т.д.
```

**На основе наших сигналов определи:**

**Desired Outcome:** "Русскоязычные пользователи ведут учёт потреблённого медиа-контента (фильмы, сериалы, книги, аниме, манга) в едином приложении"

**Opportunities (возможности)** — не менее 4:
1. Импортозамещение западных трекеров
2. Единый учёт контента из разных платформ
3. Социальные функции (достижения, сравнения, рекомендации)
4. Интеграция с российскими стримингами/библиотеками

**Для каждой Opportunity:**
- Опиши суть возможности
- Предложи 2-3 Solutions (решения)
- Укажи риски/ограничения

### Шаг 3.2: Jobs-to-be-done (JTBD)

Определи 5-7 ключевых Jobs (работ), которые пользователи хотят сделать:

Формат:
```
Job #1: [Название]
- Когда: [ситуация]
- Я хочу: [желание]
- Чтобы: [мотивация]
- Успех: [что значит сделать хорошо]
- Провалы: [что может пойти не так]
```

Примеры контекста:
- Пользователь смотрит аниме на Кинопоиске, но не может отследить всё
- Пользователь читает книги в Литрес, но не ведёт учёт
- Аниме-фанат хочет рекомендации, но не нашёл нормального русского трекера

### Шаг 3.3: Super Product Concept

На основе OST и JTBD сформулируй **Super Product Concept**:

Формат:
```
Название: [название продукта или "Без названия"]

Описание (2-3 предложения):
[Краткое описание продукта]

Ключевые возможности (3-5):
1. [Возможность 1]
2. [Возможность 2]
...

Для кого:
- [Сегмент 1]
- [Сегмент 2]

Почему сейчас:
- [Почему именно сейчас это нужно]

Барьеры:
- [Барьер 1]
- [Барьер 2]
```
