# ШАГ 4.1: PROOF OF CONCEPT (PoC)

## Определение

> PoC — минимальная демонстрация технической реализуемости основной идеи. Срок: 2-4 недели. Цель: ответить на ключевой технический вопрос.

## Ключевые технические вопросы

| # | Вопрос | Почему важен | Как проверим |
|---|--------|--------------|--------------|
| 1 | Работает ли TMDB API для поиска фильмов? | Основа функциональности | Успешный поиск 10 фильмов |
| 2 | Работает ли Open Library API для книг? | Основа функциональности | Успешный поиск 10 книг |
| 3 | Можно ли за неделю собрать базовый прототип? | Оценка скорости разработки | Прототип готов за 7 дней |
| 4 | Хватит ли бесплатных API для MVP? | Budget constraints | 3+ API работают стабильно |

## Технический стек PoC

| Компонент | Выбор | Почему |
|-----------|-------|--------|
| **Язык** | Python 3.10+ | Быстро, есть all needed libraries |
| **API фильмы** | TMDB API (бесплатный tier) | 400K+ фильмов, хорошая документация |
| **API книги** | Open Library API (бесплатный) | 20M+ книг, открытый |
| **UI** | HTML + Bootstrap | Быстро, не нужно React на этом этапе |
| **Хранение** | SQLite | Просто, не нужно поднимать PostgreSQL |

## План PoC

```
НЕДЕЛЯ 1: Техническая проверка API
├── Понедельник: Регистрация в TMDB, получение API key
├── Вторник: Python script → TMDB search (10 фильмов)
├── Среда: Open Library search (10 книг)
├── Четверг: Проверка rate limits, обработка ошибок
└── Пятница: Документация API + выводы

НЕДЕЛЯ 2: Базовый UI прототип
├── Понедельник: HTML страница с формой поиска
├── Вторник: Интеграция с Python backend (Flask/FastAPI)
├── Среда: Отображение результатов поиска
├── Четверг: Стилизация Bootstrap
└── Пятница: Демо + документирование

ИТОГО: 2 недели, ~80-100 часов dev time
```

## Результат PoC

| Метрика | Результат |
|---------|-----------|
| **Что получим** | Рабочий прототип поиска фильмов/книг |
| **Срок** | 2 недели (14 дней) |
| **Бюджет** | ~80-100 часов разработки (или ~$2-4K если нанимать) |
| **Риск** | Низкий — используем проверенные API |

## Критерии успеха PoC

| Критерий | Целевое значение | Как измерим |
|----------|-----------------|-------------|
| TMDB API работает | Да, поиск возвращает результаты | 10/10 запросов успешны |
| Open Library API работает | Да, поиск возвращает результаты | 10/10 запросов успешны |
| UI загружается | Да, страница открывается | Browser test |
| Поиск работает | Да, результаты отображаются | Manual test |
| Rate limits не блокируют | Нет критических лимитов | 100 запросов за час |

## Риски PoC и митигации

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| TMDB изменит API | Низкая | Документация свежая, backward compatible |
| Open Library нестабилен | Средняя | Есть альтернативы (Google Books API) |
| Rate limits недостаточны | Низкая | TMDB allows 50 requests/sec |
| UI не работает на мобильных | Низкая | Bootstrap responsive |

## Определение

> Prototype — полноценный UX mockup с пользовательскими flow. Срок: 5-10 недель. Цель: протестировать UX с реальными пользователями.

## Цели Prototype

1. **Валидировать UX** — пользователи понимают как пользоваться
2. **Определить критические фичи** — что должно быть в MVP

# ШАГ 4.3: MVP (Minimum Viable Product)

## Определение

> MVP — работающий продукт с минимальным функционалом для early adopters. Срок: 10-14 недель. Цель: получить 50-100 пользователей и проверить retention.

## MVP Scope

### Обязательно (Must Have)

| Фича | Описание | Приоритет |
|------|----------|-----------|
| **Регистрация/Вход** | Email + пароль, JWT auth | 🔴 Critical |
| **Добавление фильма** | Поиск TMDB, сохранение | 🔴 Critical |
| **Добавление книги** | Поиск Open Library, сохранение | 🔴 Critical |
| **Просмотр списка** | Фильмы и книги пользователя | 🔴 Critical |
| **Оценка 1-10** | Звёзды, 1-10 | 🔴 Critical |
| **Статус** | Просмотрено/В планах/Бросил | 🔴 Critical |
| **Статистика** | Количество за год | 🟡 Important |

### Желательно (Should Have)

| Фича | Описание | Приоритет |
|------|----------|-----------|
| **Аниме/Манга** | Поиск SAM DB | 🟡 Important |
| **Сериалы** | Текущий эпизод/сезон | 🟡 Important |
| **Заметки** | Текстовые заметки к элементам | 🟡 Important |
| **Цели** | Цели на год, трекинг | 🟢 Nice to have |

### Не в MVP (Out of Scope)

- Социальные функции (друзья, сравнения)
- Рекомендации (collaborative filtering)
- Интеграция с VK/Telegram
- Мобильное приложение
- Экспорт данных

## Технический стек MVP

| Компонент | Выбор | Почему |
|-----------|-------|--------|
| **Frontend** | React 18 + Vite | Быстро, modern, большое community |
| **Styling** | Tailwind CSS | Быстрая разработка, consistent |
| **Backend** | FastAPI (Python 3.10+) | Быстрый, auto-docs, Pydantic |
| **Database** | PostgreSQL | Надёжно, масштабируемо |
| **Auth** | JWT + bcrypt | Стандарт, безопасно |
| **API фильмы** | TMDB API | Бесплатно, good docs |
| **API книги** | Open Library | Бесплатно, открытый |
| **Deploy** | Railway / Render | Free tier available |

## Архитектура MVP

```
┌─────────────────────────────────────────────────────────┐
│                     FRONTEND (React)                    │
│  SPA, Tailwind CSS, React Router                         │
│  State: React Query (server state), Zustand (UI state)  │
└─────────────────────────┬───────────────────────────────┘
                          │ HTTP/REST (JWT Bearer)
┌─────────────────────────▼───────────────────────────────┐
│                    BACKEND (FastAPI)                    │
│  ├── Auth endpoints (register, login, me)              │
│  ├── Films CRUD (list, create, update, delete)         │
│  ├── Books CRUD                                        │
│  └── TMDB/OpenLibrary proxy                            │
└─────────────────────────┬───────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────┐
│                    DATABASE (PostgreSQL)                │
│  ├── users (id, email, password_hash, created_at)       │
│  ├── films (user_id, title, year, rating, status, ...) │
│  └── books (user_id, title, author, rating, status, ...)│
└─────────────────────────────────────────────────────────┘
```

## Структура данных MVP

### Users

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Films

```sql
CREATE TABLE films (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  tmdb_id INTEGER,
  title VARCHAR(500) NOT NULL,
  title_en VARCHAR(500),
  year INTEGER,
  poster_url TEXT,
  description TEXT,
  director VARCHAR(500),
  rating INTEGER CHECK (rating >= 1 AND rating <= 10),
  status VARCHAR(50) DEFAULT 'planned',
  notes TEXT,
  watched_date DATE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Books

```sql
CREATE TABLE books (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  openlib_id VARCHAR(100),
  title VARCHAR(500) NOT NULL,
  author VARCHAR(500),
  year INTEGER,
  cover_url TEXT,
  description TEXT,
  pages INTEGER,
  rating INTEGER CHECK (rating >= 1 AND rating <= 10),
  status VARCHAR(50) DEFAULT 'planned',
  notes TEXT,
  read_date DATE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
