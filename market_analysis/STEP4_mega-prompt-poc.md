## ПОШАГОВЫЙ ПРОЦЕСС

### Шаг 4.1: Proof of Concept (PoC)

**Определение PoC:**
> PoC — это минимальная демонстрация, что основная идея технически реализуема. Занимает 2-4 недели. Цель — ответить на ключевой технический вопрос.

**Для нашего продукта ключевые вопросы:**
1. Работает ли TMDB API для поиска фильмов?
2. Работает ли Open Library API для книг?
3. Можно ли собрать прототип за 2 недели?

**Формат PoC:**
```
Название: [название]
Срок: [2-4 недели]
Стек: [технологии]
Бюджет: [стоимость]
Результат: [что получим на выходе]
Метрики успеха: [как проверим]
Риски: [что может пойти не так]
```

---

### Шаг 4.2: Prototype

**Определение Prototype:**
> Prototype — это полноценный UX mockup с пользовательскими flow. Занимает 5-10 недель. Цель — протестировать UX с реальными пользователями.

**Для нашего продукта:**
- Figma с полным flow: регистрация → добавление фильма → просмотр статистики
- Минимум 3 экрана: Home, Add Item, Profile/Stats
- Тест на 10 пользователях

**Формат Prototype:**
```
Название: [название]
Срок: [5-10 недель]
Стек: [Figma + инструменты]
Бюджет: [стоимость]
Результат: [что получим на выходе]
Метрики успеха: [как проверим]
User Flow: [описание]
```

---

### Шаг 4.3: MVP (Minimum Viable Product)

**Определение MVP:**
> MVP — это работающий продукт с минимальным функционалом для early adopters. Занимает 10+ недель. Цель — получить 50-100 пользователей и проверить retention.

**Для нашего продукта MVP:**
- Backend: FastAPI + PostgreSQL
- Frontend: React SPA
- Auth: JWT + bcrypt
- Контент: Фильмы + Книги (через API)
- Метрика: retention 30+ дней

**Формат MVP:**
```
Название: [название]
Срок: [10-14 недель]
Стек: [технологии]
Бюджет: [стоимость (dev time)]
Результат: [что получим на выходе]
Метрики успеха: [как проверим]
Дорожная карта: [что делаем и когда]
```
